# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.1.x   | Yes       |
| < 1.1   | No        |

## Reporting a Vulnerability

If you discover a security vulnerability in SlideSherlock, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities.
2. Email **sachinkg12@gmail.com** with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
3. You will receive an acknowledgment within 48 hours.
4. A fix will be developed and released as a patch version.

## Scope

- The SlideSherlock pipeline and API server
- Docker configuration and deployment scripts
- Dependencies listed in `requirements.txt`

## Out of Scope

- Third-party LLM APIs (OpenAI, Ollama, etc.) — report to those providers directly
- The Library of Congress test corpus (public domain data, not a SlideSherlock component)
